Journal-oriented software checklist
===================================

The repository is structured to support common software-journal expectations:

- accessible source code;
- explicit open-source license;
- installation instructions;
- executable example or synthetic smoke test;
- user documentation;
- contribution and support guidance;
- versioned releases;
- continuous integration;
- citation metadata;
- archived DOI;
- availability and implementation statement;
- disclosure of limitations;
- no confidential data in the public repository.

Before submission
-----------------

- replace every ``Parygithub`` placeholder;
- add ORCID identifiers;
- confirm the license;
- make the repository public or provide reviewer access;
- create a tagged release;
- archive the release with Zenodo;
- verify a clean installation in a new environment;
- verify all main app functions using non-confidential data;
- provide representative screenshots;
- add an example dataset or a documented public-data route;
- state the exact version used in the manuscript.
