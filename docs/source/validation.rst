Validation and testing
======================

Automated smoke test
--------------------

``tests/smoke_test.py`` creates synthetic FIDs, runs the preprocessing
functions, aligns synthetic clinical metadata, calculates EDA outputs,
screens possible outliers, and trains the supported classifiers.

Run the smoke test with:

.. code-block:: bash

   python tests/smoke_test.py

Synthetic demonstration-data validation
---------------------------------------

``tests/test_demo_data.py`` validates the synthetic demonstration downloads
generated by the application.

Run the demonstration-data test with:

.. code-block:: bash

   python tests/test_demo_data.py

The test confirms that the generated demonstration data contain:

- 16 ``fid`` files;
- 16 ``acqus`` files;
- 16 unique FID SHA-256 hashes;
- 16 clinical metadata rows;
- unique clinical ``study_id`` values;
- exact agreement between the clinical ``study_id`` values and the NMR
  sample-folder names.

Continuous integration
----------------------

GitHub Actions workflows:

- compile all Python modules;
- run the smoke test;
- run the synthetic demonstration-data validation test;
- test the application on Python 3.10, 3.11, and 3.12;
- build the Sphinx documentation with warnings treated as errors.

Scientific validation still required
------------------------------------

Automated tests establish software functionality and internal consistency.
They do not establish biological, analytical, diagnostic, or clinical
validity.

A scientific publication or applied study should additionally consider:

- comparison with a trusted or independently implemented processing workflow;
- parameter-sensitivity analysis for important preprocessing decisions;
- independent, external, or appropriately nested validation for supervised
  models;
- manual review of representative raw and processed spectra;
- assessment of batch effects, acquisition drift, and instrument-related
  variation;
- checks for sample-ID, class-label, and clinical-metadata consistency;
- evaluation of missing data, class imbalance, and potential data leakage;
- reporting of all preprocessing, outlier-handling, and model-selection
  decisions;
- interpretation by researchers with appropriate NMR, statistical, and
  clinical expertise.

Research-use limitation
-----------------------

NMRMetaboWizard is a research software tool. It is not a validated medical
device, diagnostic system, or substitute for expert review. Synthetic
demonstration data are provided only to test the software workflow and must
not be used for biological or clinical interpretation.
