Machine learning(ML)
================

Predictor sources
-----------------

- NMR only
- Clinical only
- NMR + clinical
- PSA only

Models
------

- logistic regression;
- random forest;
- linear SVM using ``LinearSVC``;
- ANN/MLP using ``MLPClassifier``.

Holdout evaluation
------------------

The default test fraction is 0.25. The split is stratified when class counts
permit. The app reports training and test accuracy and balanced accuracy,
classification reports, confusion matrices, and individual predictions.

Cross-validation
----------------

Five-fold stratified cross-validation is enabled by default and can be
disabled. The number of folds is reduced automatically when the smallest class
contains fewer samples than requested.

Leakage control
---------------

Median imputation, standardization, and optional PCA are inside the
scikit-learn ``Pipeline``. These transformations are fitted separately in each
training split or cross-validation fold.

Optional PCA
------------

PCA is off by default. When enabled, the metrics distinguish:

- input features before PCA;
- features seen by the classifier after PCA.

Probability and ROC outputs
---------------------------

Logistic regression, random forest, and ANN provide ``predict_proba`` and can
produce probability and ROC plots after a valid holdout fit.

The current linear SVM uses ``LinearSVC``. It does not provide calibrated
probabilities, so predicted-probability and ROC tables are not generated by
the current implementation.

Feature importance
------------------

- logistic regression and linear SVM: mean absolute coefficient;
- random forest: impurity-based feature importance;
- ANN: mean absolute first-layer weight, which is an approximate heuristic.

When optional PCA is used, the displayed feature-importance table is obtained
from a separate full-data model without PCA so original variables remain
visible. It must therefore be interpreted as an exploratory explanation, not
as a held-out performance estimate.

Model history
-------------

Each model run is appended to a session history table with settings and
metrics. Download the table before closing or restarting the app.
