# Software-paper submission checklist

## Repository

- [ ] Public GitHub repository or confidential reviewer access
- [ ] Open-source license confirmed
- [ ] Tagged release corresponding to manuscript
- [ ] Zenodo DOI for the tagged release
- [ ] Installation tested in a clean environment
- [ ] Documentation website builds successfully
- [ ] Synthetic/public example data available
- [ ] No confidential or identifiable data in history
- [ ] Tests and continuous integration pass
- [ ] Changelog and citation metadata updated

## Manuscript

- [ ] Software name included in title
- [ ] Statement of need and comparison with related tools
- [ ] Design and implementation described
- [ ] Scientific methods described accurately
- [ ] Case-study data and ethics statements complete
- [ ] Availability and implementation statement contains URLs and DOI
- [ ] Data availability statement complete
- [ ] License and version stated
- [ ] Limitations clearly described
- [ ] Figures readable at journal size
- [ ] Supplementary user documentation cited

## Validation

- [ ] Core functions tested
- [ ] Reviewer can install and run software
- [ ] Representative input/output demonstrated
- [ ] Parameter defaults documented
- [ ] Leakage controls described
- [ ] Model validation design justified
- [ ] Independent validation or limitations stated
